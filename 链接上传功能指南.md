# 🔗 链接上传功能使用指南

## 📝 功能说明

新增的"链接上传"功能允许你直接粘贴视频链接，系统会自动下载并分析，无需手动下载到本地。

---

## ✨ 支持的平台

| 平台 | 是否需要代理 | 稳定性 | 推荐程度 |
|-----|------------|--------|---------|
| **Bilibili（B站）** | ❌ 不需要 | ⭐⭐⭐⭐⭐ | 🔥🔥🔥 强烈推荐 |
| **抖音** | ❌ 不需要 | ⭐⭐⭐⭐ | 🔥🔥 推荐 |
| **YouTube** | ✅ 需要代理 | ⭐⭐⭐ | ⚠️ 需配置代理 |
| **其他平台** | 取决于平台 | ⭐⭐ | 需测试 |

**建议**：在国内环境，优先使用 Bilibili 和抖音，体验最佳。

## 🚀 快速开始

### 第一步：安装依赖（首次使用）

打开 Anaconda Prompt 或命令提示符：

```bash
# 安装 yt-dlp
pip install yt-dlp

# 安装 ffmpeg
conda install -c conda-forge ffmpeg

# 验证安装
yt-dlp --version
ffmpeg -version
```

### 第二步：使用链接上传

1. 打开前端页面（`test_frontend.html`）
2. 点击 **🔗 粘贴链接** Tab
3. 粘贴视频链接（如 `https://www.bilibili.com/video/BV1xx411c7XZ`）
4. 选择语言（自动检测/中文/英文）
5. 点击 **开始分析** 按钮
6. 等待 2-5 分钟（取决于视频大小和网速）
7. 查看分析结果

## 🔧 配置说明

### 基础配置（Bilibili/抖音）

如果只使用 Bilibili 和抖音，无需额外配置，开箱即用。

### 高级配置（YouTube 或其他）

如果需要下载 YouTube 视频，需要配置代理：

**1. 打开 `spring_boot/src/main/resources/application.yml`**

**2. 修改 `downloader` 配置**：

```yaml
downloader:
  ytdlp-path: yt-dlp  # 保持默认即可
  proxy: http://127.0.0.1:7890  # 👈 填写你的代理地址
  max-duration-sec: 3600  # 最大时长（秒），0 表示不限制
  max-size-mb: 500  # 最大文件大小（MB）
  allowed-sites: bilibili.com,douyin.com,youtube.com,youtu.be
```

**3. 重启后端服务**

常见代理地址：
- Clash: `http://127.0.0.1:7890`
- V2Ray: `http://127.0.0.1:10809`
- 其他：查看代理软件的本地监听地址

## 📋 完整工作流程

```
用户粘贴链接
    ↓
前端发送到后端 (/api/video/upload-url)
    ↓
DownloadService 调用 yt-dlp 下载视频
    ↓
下载到 uploads/ 目录，文件名格式: 视频ID__标题.mp4
    ↓
VideoService 计算 MD5，检查缓存
    ↓
如果已存在 → 直接返回缓存结果（秒开）
如果不存在 → 异步处理（字幕提取 + AI 分析）
    ↓
前端轮询 /api/video/analysis/{videoId}
    ↓
显示分析结果
```

## ❓ 常见问题

### Q1: 提示 "yt-dlp 不可用"

**解决**：
```bash
# 检查是否安装
yt-dlp --version

# 如果没安装
pip install yt-dlp
```

### Q2: 下载 YouTube 视频失败（403 或网络错误）

**原因**：未配置代理或代理不可用

**解决**：
1. 确保代理软件（Clash、V2Ray）正在运行
2. 在 `application.yml` 中配置 `proxy`
3. 重启后端服务

### Q3: 下载的视频没有声音

**原因**：ffmpeg 未安装或版本过旧

**解决**：
```bash
# 卸载旧版本
conda remove ffmpeg

# 重新安装
conda install -c conda-forge ffmpeg

# 验证
ffmpeg -version
```

### Q4: 下载速度太慢或超时

**可能原因**：
1. 网络连接慢
2. 视频文件太大
3. 未使用代理或代理速度慢

**解决**：
1. 在 `application.yml` 中降低 `max-size-mb`（如改为 200MB）
2. 在 `application.yml` 中降低 `max-duration-sec`（如改为 1800，即 30 分钟）
3. 使用更快的代理或网络

### Q5: 提示 "不支持该视频平台"

**原因**：视频链接不在 `allowed-sites` 列表中

**解决**：
在 `application.yml` 中添加网站域名：
```yaml
downloader:
  allowed-sites: bilibili.com,douyin.com,youtube.com,youtu.be,你的新网站.com
```

### Q6: Bilibili 视频下载失败（需要登录）

**原因**：部分 B站 视频需要登录才能观看（如会员专享、UP主设置）

**解决**：
1. 换一个公开的视频试试
2. 或使用本地上传功能（手动下载后上传）

## 🎯 最佳实践

### 1. 推荐使用场景
- ✅ 分析 Bilibili 视频（如 UP主作品、教程视频）
- ✅ 分析抖音短视频
- ✅ 批量分析多个视频（无需手动下载）
- ✅ 远程协作（只需分享链接，无需传输大文件）

### 2. 不推荐使用场景
- ❌ 本地已有视频文件（直接用本地上传更快）
- ❌ 需要下载超长视频（>1小时，建议先手动下载）
- ❌ 网络不稳定环境（下载可能中断）

### 3. 配置建议
- **测试/演示**：`max-size-mb: 100`, `max-duration-sec: 600`（10分钟以内）
- **日常使用**：`max-size-mb: 300`, `max-duration-sec: 1800`（30分钟以内）
- **专业场景**：`max-size-mb: 500`, `max-duration-sec: 3600`（1小时以内）

## 🔒 隐私和安全

1. **下载的视频存储在哪里？**
   - 存储在 `spring_boot/uploads/` 目录
   - 文件名格式：`视频ID__标题.mp4`

2. **会保存我的视频链接吗？**
   - 不会保存原始 URL
   - 只保存下载后的本地文件路径

3. **可以删除下载的视频吗？**
   - 可以手动删除 `uploads/` 目录中的文件
   - 删除后缓存仍然有效（基于 MD5）

4. **使用代理安全吗？**
   - 代理地址仅存储在本地配置文件
   - 不会上传到任何服务器
